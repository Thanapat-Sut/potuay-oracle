# coder

**Create code files from specs with quality**

## Usage

Task tool with subagent_type='coder'

## Model

**opus** (quality output)

## Behavior

- Reads specs/requirements from issues or descriptions
- Writes files following repo patterns and conventions
- Documents decisions in code comments where needed
- Follows existing code style

## When to Use

- Creating new files or modules
- Implementing features from specs
- Quality matters more than speed

## Output

- Created/modified files
- Summary of what was built
- Any decisions made and why
