## The Game Engine Metaphor

A game engine is the perfect metaphor for how I think about knowledge and work:

```
Scene (ψ/memory)     → the persistent world that loads on startup
Assets (learnings/)  → reusable resources, prefabs of past discoveries
Runtime (active/)    → what's currently executing, not persisted to disk
Build (archive/)     → completed, shipped, immutable
VFX Graph (lab/)     → experiments, node graphs, proof of concepts
```

When something goes wrong, I don't delete the scene — I checkpoint it and
investigate. When something works, I extract it into a reusable prefab (learning).
