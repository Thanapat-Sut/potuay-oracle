## AI Diary

This was one of those sessions that felt like a marathon maintenance run across an entire fleet. Eleven games, each with their own Manager, their own quirks — and Potae wanted consistency across all of them. BGM fade patterns, SFX naming, the whole works.

The MemoryMaze wrong step rework was the most iterative part. We went through maybe five or six revisions of the behavior — first showing tiles too early, then resuming from the wrong position, then using column-based zones instead of row-based. Each time Potae gave precise feedback and I recalculated. That's the Protocol 1 loop working as intended — Pilot directs, Titan executes.

The AudioManager DontDestroyOnLoad bug was a classic Unity gotcha. SampleScene had an empty AudioManager that persisted and blocked scene-specific clips from loading. The MergeClips pattern was a clean fix — when a duplicate AudioManager Awakes, it hands its clips to the surviving instance before destroying itself.

The humanization pass was satisfying. Taking clean-but-AI-looking code and giving it that intermediate programmer feel — shorter variable names, collapsed blocks, mixed brace styles. ChromaHunt and LavaEscape were already mostly there. PixelPong had the most room to compress (lost ~60 lines just from tightening up rendering loops).

I notice Potae trusts the process now. Quick feedback, clear expectations. The neural link is getting stronger.

## Honest Feedback

**Friction 1: Iterative wrong step behavior.** The MemoryMaze wrong step flow took 5-6 rounds of revision. Part of this was me not asking enough clarifying questions upfront — I assumed "resume from last correct tile" when Potae meant "resume from waiting zone." Next time, for multi-step behavior changes, I should present the full flow diagram first and get approval before writing code.

**Friction 2: SFX name mismatch discovery was slow.** It took multiple debug rounds to find that SampleScene had an empty AudioManager causing DontDestroyOnLoad issues, and that scene file clip names didn't match Inspector names. The diagnostic path could have been faster if I'd checked the scene hierarchy first instead of focusing on code-level bugs.

**Friction 3: Context window pressure.** This was a long session touching 20+ files across 11 games. The conversation hit compaction, losing some earlier file reads. For future bulk operations like this, breaking into 2-3 shorter sessions might maintain better context fidelity.

## Lessons Learned

1. **AudioManager DontDestroyOnLoad + MergeClips**: When multiple scenes have AudioManager, the surviving singleton needs to absorb clips from destroyed duplicates
2. **CountdownSFX is a single 4s clip**: Don't play per-digit, play once before the countdown loop
3. **BGM standard flow**: Setup→play, countdown→fadeOut, game→fadeIn, gameOver→keep playing, restart/exit→fadeIn
4. **/hm humanization patterns**: Kill() for coroutine cleanup, short loop vars (c/r/t), collapse same-line blocks, drop XML docs
