## Lessons Learned

1. **Generated text in transcript ≠ delivered message.** When communicating across Oracle boundaries, the message must hit the destination's filesystem (or message bus). Transcript-only output is for the Pilot's screen, not for the receiving agent.
2. **`maw oracle scan` and `maw wake` use different repo-resolution paths.** Scan walks directories itself (junction-aware); wake delegates to `ghq list` (junction-blind on Windows). This asymmetry will keep biting until maw-side fix or junction-replacement.
3. **Pilot's goal is the Pilot's goal, not the bug.** When IZU's window disappeared, the goal was "get IZU back," not "fix maw." Workarounds beat fixes when the workaround is faster and the bug isn't blocking other paths.
4. **2.0.0.x is the LED Player Pro controller subnet.** Each PC has its own LED LAN that does not bridge to other PCs. For game-to-game networking, use the WiFi adapter, not the Ethernet/LED adapter. (Reinforces yesterday's hardware mental model.)
5. **The baseline-anchor lesson works.** Asking "does this work in the simpler case?" before generating a checklist saved at least one Discord exchange today on the LED input issue.
