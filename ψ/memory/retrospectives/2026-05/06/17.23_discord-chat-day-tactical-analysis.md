## Lessons Learned

1. **Anchor diagnosis to the user's baseline-working artifact, not to the failing artifact.** When something works elsewhere, the suspect list narrows immediately. The first question for any "X doesn't work" is "what's working that's similar?"
2. **Read Claude-authored commit messages first.** They explain intent and structure better than diffs. Only dive into diffs for surprises.
3. **`Color32` with hex literals is more readable than `Color` with float when the spec is hex.** Convert only when math needs floats.
4. **"No, this needs scripts" is a complete answer.** Don't manufacture config-only paths to please. Outline what the change would look like, let the Pilot decide.
